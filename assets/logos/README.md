# Brand logos — single source of truth

This folder is the **canonical home** for the BimDossier logo. Nothing in the codebase
embeds the logo as base64 or keeps its own copy — every app references these files.

## The in-app logo (used everywhere `<BrandMark>` renders)

| File | What | Use it when |
|------|------|-------------|
| `brand-primary.png` | the **blue** "A-house" mark | the background is light (default — most surfaces) |
| `brand-white.png`   | the **white** mark         | the background is primary blue (auth hero, sidebars, marketing brand panel) |

### How it reaches each app

`scripts/sync-brand-assets.mjs` copies both files into:

- `apps/web/public/brand/` and `apps/portal/public/brand/` — **gitignored**, auto-copied on
  `predev` / `prebuild` (the same hook pattern as the WASM assets). The shared
  `<BrandMark variant="primary|white">` component (`packages/brand`) loads them from the
  root-relative URL `/brand/brand-<variant>.png`.
- `apps/mobile/assets/images/` — **committed** (React Native / Metro must bundle real files).
  `apps/mobile/src/components/BrandMark.tsx` `require()`s them.

### To swap the logo

1. Drop a replacement PNG (transparent background) at `brand-primary.png` and/or `brand-white.png`.
2. Run `node scripts/sync-brand-assets.mjs` (web/portal also do this automatically on next dev/build).
3. For mobile, commit the regenerated `apps/mobile/assets/images/brand-*.png`.

No code changes needed — every surface picks up the new art.

## Not covered here (separate pipeline)

Browser favicons (`apps/{web,portal}/public/favicon*`, `apple-icon.png`) and native app
icons / splash screen (`apps/mobile/assets/images/icon.png`, `android-icon-*`, `splash-icon.png`)
are generated by `scripts/generate-logo-assets.mjs` and still use the **old** mark. They are a
deliberate later step (full rebrand), not part of the in-app logo swap.
