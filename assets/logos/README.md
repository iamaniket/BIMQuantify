# Brand logos — single source of truth

This folder is the **canonical home** for the BimDossier logo. Nothing in the codebase
embeds the logo as base64 or keeps its own copy — every app references these files.

## The in-app logo (used everywhere `<BrandMark>` renders)

| File | What | Use it when |
|------|------|-------------|
| `brand-primary.png` | the **blue** "A-house" mark | the background is light (default — most surfaces) |
| `brand-white.png`   | the **white** mark         | the background is primary blue (auth hero, sidebars, marketing brand panel) |

### How it reaches each app

`scripts/sync-brand-assets.mjs` copies both files into:

- `apps/web/public/brand/` and `apps/portal/public/brand/` — **gitignored**, auto-copied on
  `predev` / `prebuild` (the same hook pattern as the WASM assets). The shared
  `<BrandMark variant="primary|white">` component (`packages/brand`) loads them from the
  root-relative URL `/brand/brand-<variant>.png`.
- `apps/mobile/assets/images/` — **committed** (React Native / Metro must bundle real files).
  `apps/mobile/src/components/BrandMark.tsx` `require()`s them.

### To swap the logo

1. Drop a replacement PNG (transparent background) at `brand-primary.png` and/or `brand-white.png`.
2. Run `node scripts/sync-brand-assets.mjs` (web/portal also do this automatically on next dev/build).
3. For mobile, commit the regenerated `apps/mobile/assets/images/brand-*.png`.

No code changes needed — every surface picks up the new art.

## The browser favicons (separate pipeline)

Browser-tab favicons and the apple-touch-icon have their own source art and generator:

| File | What | Feeds |
|------|------|-------|
| `fevi-primary.png` | the **blue** "building" mark | light browser tabs |
| `fevi-white.png`   | the **white** "building" mark | dark browser tabs + the apple-icon blue tile |

Run `node scripts/generate-favicons.mjs` to regenerate, for **both** `apps/web` and
`apps/portal`, `public/{favicon.svg, favicon-light.png, favicon-dark.png, favicon.ico, apple-icon.png}`.
The **apple-icon is the exception** — it uses the full in-app brand logo (`brand-primary.png`)
on a solid white tile, not the flat `fevi-*` building mark, since a home-screen icon reads
better as the real logo. Everything else uses the `fevi-*` marks.
The theme switch (blue mark on light tabs, white on dark) follows the OS `prefers-color-scheme`;
`favicon.svg` embeds both marks and toggles via an in-SVG media query (the only thing Chrome
honors). Wiring is in each app's `src/app/layout.tsx` `metadata.icons`. To swap the favicon,
drop replacement transparent PNGs at `fevi-primary.png` / `fevi-white.png` and rerun the script.

## Not covered here (separate pipeline)

Native app icons / splash screen (`apps/mobile/assets/images/icon.png`, `android-icon-*`,
`splash-icon.png`) are generated by `scripts/generate-logo-assets.mjs` and still use the **old**
mark. They are a deliberate later step (full rebrand), not part of this favicon swap.
